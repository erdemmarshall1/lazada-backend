18. Fixed "Not allowed nest placeholder" Error (vue-i18n message compiler)
    - Root cause: profitLabel translations in en.json had ${{retailPrice}} which the
      message compiler's tokenizer misinterpreted as nested placeholder when braceNest
      was already active from the preceding {{percentage}} placeholder
    - Fix: Removed $ sign from translation strings (en.json lines 818, 823), prepended
      $ to retailPrice value at the template call site instead
    - Files modified:
      - frontend/src/locales/en.json (profitLabel translations)
      - frontend/src/views/wholesale/SourceGoods.vue (line 33)
      - frontend/src/views/wholesale/SourceGoodsDetail.vue (line 11)

19. Fixed NOT_ALLOW_NEST_PLACEHOLDER Across All Locales
    - Root cause: ALL locale files used {{param}} (double-brace) syntax which triggers
      NOT_ALLOW_NEST_PLACEHOLDER in vue-i18n v9 message compiler's tokenizer (every { increments
      braceNest, second { in {{ throws when braceNest >= 1)
    - Fix: Replaced all {{param}} → {param} (correct v9 syntax) across all 20 locale JSON files
    - Parameters affected: year, stock, categoryTitle, amount, price, percentage, retailPrice
    - No template changes needed ($t() calls already pass named parameters correctly)
    - Build: 2248 modules, 0 errors

20. Fixed "Transaction not found" on Admin Deposit Approval
    - Root cause: Field name mismatch — frontend sends { transactionId: id } but backend
      reads const { id } = req.body → findById(undefined) → null → "Transaction not found"
    - Fix: Changed walletController.js adminApproveTransaction (line 224) and
      adminRejectTransaction (line 254) to read const { id, transactionId } = req.body
      and use const txId = id || transactionId
    - Files modified: backend/controllers/walletController.js
